PRO_API pro_error pro_become ( pro_state_ref  ,
pro_ref  actor,
pro_ref  new_beh 
)

Specify the behavior for an actor.

Essentially sets up message forwarding from the old actor to the new actor. Usually, 'become' forwards messages to a newly created actor but this is not required.

Parameters:
actorThe actor we a specifying the new behavior for.
new_behThe actor that will recieve all messages from the old actor.
Returns:
PRO_OK if successful PRO_INVALID_STATE if either actor or new_beh are not valid actors.
