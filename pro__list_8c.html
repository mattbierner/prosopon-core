Appends an value to a list to create a new list.

Does not modify the origin list.

Parameters:
[out]new_msgA reference to the new list with the new object appended with a reference count of 1. Should not be the same as msg, this will cause a memork leak.
Returns:
PRO_OK if successful PRO_INVALID_STATE if msg does not reference a list or value does not reference a valid object.
